Leaky abstraction

In software development, a leaky abstraction is an abstraction that leaks details that it is supposed to abstract away. As coined by Joel Spolsky, the Law of Leaky Abstractions states: All non-trivial abstractions, to some degree, are leaky. In other words, a leaky abstraction is one which is unsuccessful in hiding all the messy detail’s it’s meant to cover up. Example(s) One real-world example is a printed bus schedule. Three pulls

Three pulls is a production / efficient Pottery technique for wheel throwing in which one pulls the form to its full height in only three pulls: Normal pull; pull as you normally would using your fingers or a sponge, etc Normal pull Rib pull; pull up with a wooden/etc rib on the outside In this way you pull and remove excess slip in the same movement.
